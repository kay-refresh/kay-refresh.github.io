jQuery的ready不等于JavaScript的load

2017-03-21

jQuery 定义的 ready 事件与 JavaScript 定义的 load 时间都表示页面 初始化行为，但是它们之间并非完全相同。

HTML DOM 文档加载是按顺序执行的，这与浏览器的渲染方式有关系，一般浏览器渲染操作的顺序大致按如下几个步骤：

  1. 解析 HTML 结构。
  2. 加载外部脚本和样式表文件。
  3. 解析并执行脚本代码。
  4. 构造 HTML DOM 模型。
  5. 加载图片等外部文件。
  6. 页面加载完毕。

jQuery 的 ready 与 JavaScript 的 load 二者的差异：

  1. 执行时机不同。
    load 事件必须等到网页中所有内容全部加载完毕之后，才被执行。

  • 如果一个页面中包含了大数据的多媒体文件，则会出现网页文档已经呈现出来，但因为网页数据没有完全被加载完毕，导致 load 事件不能够即时被触发。

  • 开发人员如果习惯把网页初始化配置的脚本都放在 load 事件处理函数中，会因为页面数据没有完全被加载出来，导致网页呈现和脚本初始化配置不能够保持同步，从而影响了页面的可用性。

jQuery 的 ready 事件是在 DOM 结构绘制完毕之后就执行。

  • 它先于外部文件的加载就被执行了，这样就能够确保文档呈现和脚本初始化设置保持同步。

ready 事件先于 load 事件被激活，如果网页文档中没有加载外部文件，它们的响应事件基本上是相同的。
